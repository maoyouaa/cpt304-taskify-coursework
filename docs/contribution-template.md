# Contribution Evidence Template

Use this when filling `individual-contribution.xlsx` and Section 7 of the report.

## Recommended member mapping

- **Member A**
  - deployment setup
  - Vercel evidence capture
  - Codecov / CI follow-up
- **Member B**
  - accessibility audit
  - baseline screenshots
  - Lighthouse evidence
- **Member C**
  - i18n, cookie banner, privacy page
  - report figure assembly
- **Member D**
  - persistence, validation, tests
  - coverage verification

## What each member should submit

- one branch name
- one PR link
- one short sentence describing their owned task
- one or more representative commits

## Example wording for the spreadsheet

- "Implemented accessible labelled forms and keyboard focus improvements; captured baseline and post-fix accessibility evidence."
- "Implemented persistent task storage, validation rules, and automated tests; verified coverage locally."
- "Implemented language switching, cookie consent, and privacy policy page; prepared report figures."
- "Configured deployment, verified production status, and coordinated evidence capture for baseline standards."

## Important note

Do not invent multi-person PR history that did not happen. If the repository history is partially centralized, keep the spreadsheet honest and use task ownership, commit evidence, and report authorship notes consistently.
